Hastakshar Web Application

Internship Information

  • Category: Deep Learning, Web Development
  • Github URL: Click Here

This web application will provide captions for the users who don’t know the Indian Sign Language themselves. Thus, this web application will enable convenient conversation for all parties.

One of the biggest things which has been noticed during the COVID-19 pandemic is how family and friends have been keeping in contact with the help of multiple video meeting platforms and apps. These social interactions have been so important during this period to compensate the lack of physical contact and have helped to keep spirits high during a truly difficult time for us all. COVID-19 has also forced many people to work from their homes. In-person interactions are a necessity at the workplace, without which information is lost or misinterpreted when just using written and verbal communication. However, there is a great digital divide, which was brought by this shift in technology. Those who are deprived from verbal communication, the deaf and mute, face difficulties using this technology. HastAkshar will come to their aid. HastAkshar will interpret limited ISL cues and transcribe these non-verbal cues.

Significance

  • The user can connect to another user via video call
  • Before entering into room, the user can set audio and video preferences by toggling the button provided for the same. The same functionality is provided in the room, too.
  • Once the users are inside the room using the generated 'roomId', they can access the calling interface which further provides access to chat interface and ISL Interpretation Service.
    • While within the call, the user can broadcast the message within the room. All the users and only the users within the room can access these messages
    • The users can view captions for 15 different ISL gestures including both single-handed and double-handed ones, by enabling ISL interpretation
  • While in the room, the user can invite another user in the room, by entering their user name, registered in the application

Implementation Details

    • This web application is designed using Django Framework
    • This web application used Postgres as database
    • The Video Calling Service makes use of Socket.IO for communication between the users
    • The default PeerCloud service is used for establishing connections between any two users
    • The messages are sent across by broadcasting a Socket.IO event within the room
    • All the button toggles are implemented using JavaScript
    • The Sign Language Interpretation module’s activities are primarily supported by a deep learning model which is configured using TensorFlow models. This is done by using SSD MobileNet and TensorFlow’s object detection libraries
    • Feature extraction is done using ‘ssd_mobilenet_v2_fpb_keras’ function with ‘RELU_6’ activation function
    • The custom dataset used for training this model includes 40 labeled images per ISL gesture
    • The dataset is split in 4:1 ratio for training and the latter for testing purposes, that is, 32 images are used for training and 8 images are used for testing
    • The images have been contributed by different individuals with no particular background filtering or setup and without any image data augmentation technique.
    • The images were labeled using ‘labelImg’ which is an open-source tool that generates XML files in PASCAL VOC format.
    • The model is trained until 30,000 epochs, at a learning rate of 0.0286.
    • The total loss recorded in the model training process was 0.172
    • For measuring the accuracy of the object detection model, metrics such as Recall, Precision, Mean Average Precision (mAP), and Intersection over Union (IoU) is used. Below are the tabulated results:
